# Contributing to StudyTales

Hey there! Thanks for considering contributing to StudyTales. We're a small project trying to make studying fun and rewarding through game mechanics. Every contribution helps!

## How You Can Help

### 🐛 Found a Bug?
- Check if it's already reported in Issues
- Create a detailed bug report with:
  - What you were doing
  - What you expected to happen
  - What actually happened
  - Screenshots if possible
  - Your device/browser info

### 💡 Have a Feature Idea?
- Start a Discussion first to get feedback
- Describe how it would improve the game
- Explain the benefit to players
- Be open to suggestions!

### 📝 Improving Documentation
- Fixing typos in README or docs
- Clarifying confusing sections
- Adding examples
- Translating content

### 🎨 Design & UI
- Proposing new card designs
- Suggesting UI improvements
- Creating game artwork
- Recording demo videos

### 💻 Code Contributions

#### Getting Started
1. Fork the repo
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/Aistudygamedevelopment.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test everything works
6. Commit with clear messages: `git commit -m "Add: new feature description"`
7. Push to your fork
8. Open a Pull Request

#### Code Style
- Use TypeScript for type safety
- Follow the existing code style (we have ESLint & Prettier!)
- Add comments for complex logic
- Keep components focused and modular
- Name things clearly

#### Project Structure
```
src/
├── app/
│   ├── components/        # React components for each page
│   ├── data/              # Game data (characters, etc)
│   ├── hooks/             # Custom React hooks
│   └── App.tsx            # Main app component
├── styles/                # Global CSS and theme
└── main.tsx               # Entry point
```

#### Running Locally
```bash
npm install
npm run dev
```

#### Before Submitting
- [ ] Run `npm run build` to check for errors
- [ ] Test on mobile if possible (important for a game!)
- [ ] Add comments to explain what you changed
- [ ] Update README if you changed how things work
- [ ] Keep changes focused (one feature per PR)

## Community Guidelines

- **Be kind.** Everyone is volunteering their time
- **Be clear.** Explain your ideas and reasoning
- **Be open.** Accept feedback and suggestions gracefully
- **Be patient.** We're all doing this in our free time

## Questions?

Feel free to open an Issue with the `question` label or start a Discussion. We're happy to help!
